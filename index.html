a) Create a function volumeOfSphere(r) that returns the volume of a sphere of a given radius r.

Example use:

var result = volumeOfSphere(4);
// result => 268.082573106329

Solution:

function volumeOfSphere(r) {
  return (4 / 3) * Math.PI * r ** 3;
}
              
console.log(volumeOfSphere(3)); // 113.09
console.log(volumeOfSphere(2)); // 33.51

I used the unbuilt JavaScript Math.PI static data property to calculate the volume of a sphere. I found the formula on Google as shown below.

In addition I used the The exponentiation (**) operator instead of e.g. writing r * r * r, which would be mathematically correct, but would result in less elegant code.

b) Create a function countDown(fromN) that counts down from the provided number down to zero (using console.log). If fromN is 0 or smaller, the function should instead print the message "fromN must be bigger than 0".

Example use:

countDown(4);

4
3
2
1
0
countDown(-1)

"fromN must be bigger than 0"

Solution:

function countDown(fromN) {
  if (fromN <= 0) {
    console.log('fromN must be bigger than 0');
  } else {
    for (let i = fromN; i >= 0; i--) {
      console.log(i);
    }
  }
}

Here I made sure to add the conditional if statement handling the scenario where the user inputs 0 or less first because it's best practice to not open for potential bugs where both statements resolve to true. This could have potentially happened had I chosen different operator values in the for loop, and switched around the conditional statements.

I also used an else statement, again to emphasise that there should only be one segment of code that should run, either the if, or the else. If I had used two if statements instead this would also have opened the door to potential bugs.

c) Create a function printSequence(n, m) that prints every number in a range n to m, with the following exceptions:

  • if the number is divisible by both 3 and 7, print "wubbalubba" instead of the number
  • if the number is divisible by 3, print "wubba" instead of the number
  • if the number is divisible by 7, print "lubba" instead of the number

Example use:

printSequence(1, 23)

2
wubba
4
5

11
wubba
13
lubba
wubba

Solution:

function printSequence(n, m) {
  if (n === m) {
    console.log(n);
  }
  if (n < m) {
    for (let i = n; i <= m; i++) {
      if (i % 3 === 0 && i % 7 === 0) {
        console.log('wubbalubba');
      } else if (i % 3 === 0) {
        console.log('wubba');
      } else if (i % 7 === 0) {
        console.log('lubba');
      } else console.log(i);
    }
  }
  if (n > m) {
    for (let i = m; i <= n; i++) {
      if (i % 3 === 0 && i % 7 === 0) {
        console.log('wubbalubba');
      } else if (i % 3 === 0) {
        console.log('wubba');
      } else if (i % 7 === 0) {
        console.log('lubba');
      } else console.log(i);
      m;
    }
  }
}

To solve this one I made sure to add an if statement handling the event that n and m are the same number. It's not specified in the question what should happen, but I decided to print the given number if the function receives two equal numbers as arguments.

An interesting thing to notice is that I used triple equals, also called strict equality (===) instead of the double equlity operator (==) because I wanted to make sure the user gave me the same type, and not e.g. one number and one string. I could have made sure that the input only took the number type, but I felt it was outside of the scope of what this question was asking for.

Another interesting bit of code is that I used the remainder (%) operator to check if the current index was divisible by 3, 7, or both.

An alternative way of solving this case would have been to use while loops instead of for loops. I could've also written it with switch statements which given a longer function could have improven the readability of the code.

d) Create a function holidayGreeting() that takes no arguments, which prompts for your name and alerts out a personal holiday greeting

Example use:

holidayGreeting();
"What is your name?"
=> 'Frank'
"Happy holidays Frank!"

holidayGreeting();
"What is your name?"
=> 'Anna'
"Happy holidays Anna!"

Solution:

function holidayGreeting() {
  const name = prompt('What is your name?');
  alert(`Happy holidays ${name}!`);
}

This code was relatively short and sweet, I made a constant called name which is defined as the result of the window.prompt() which opens as soon as the function runs. This name is then used in the following window.alert() which runs using template literals in order to combine the name of the person directly into the message.

If I had used regular quotes (""), or ('') without the template strings (``) the name constant wouldn't have appeared, instead what would've been alerted would be "Happy holidays name!" regardless of what the const name had saved.
